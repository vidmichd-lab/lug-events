# Руководство по внесению вклада

Спасибо за интерес к проекту! Мы рады любому вкладу.

## Процесс разработки

### 1. Форк репозитория

1. Нажмите кнопку "Fork" на странице репозитория
2. Клонируйте свой форк:
```bash
git clone https://github.com/YOUR_USERNAME/events-platform.git
cd events-platform
```

### 2. Создание ветки для фичи

```bash
git checkout -b feature/название-фичи
# или
git checkout -b fix/описание-бага
```

### 3. Внесение изменений

- Следуйте стилю кода проекта
- Добавляйте комментарии где необходимо
- Обновляйте документацию при изменении API

### 4. Коммиты

Используйте конвенциональные коммиты:

```
feat: добавлена новая функция
fix: исправлена ошибка
docs: обновлена документация
style: изменения форматирования
refactor: рефакторинг кода
test: добавлены тесты
chore: обновление зависимостей
```

### 5. Push и Pull Request

```bash
git push origin feature/название-фичи
```

Затем создайте Pull Request на GitHub.

## Стандарты кода

### Backend (Node.js)

- Используйте ESLint конфигурацию проекта
- Следуйте стилю кода (2 пробела для отступов)
- Добавляйте JSDoc комментарии для функций
- Обрабатывайте все ошибки

### Frontend (React/TypeScript)

- Используйте TypeScript для всех новых файлов
- Следуйте правилам ESLint
- Используйте функциональные компоненты с хуками
- Добавляйте типы для всех props и state

## Тестирование

Перед отправкой PR убедитесь:

- [ ] Код компилируется без ошибок
- [ ] Все существующие тесты проходят
- [ ] Добавлены тесты для новой функциональности
- [ ] Линтер не выдает ошибок

## Pull Request

При создании PR укажите:

1. Описание изменений
2. Связанные issues (если есть)
3. Скриншоты (для UI изменений)
4. Чеклист выполненных задач

## Вопросы?

Создайте Issue с вопросом или предложением.
